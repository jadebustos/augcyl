2.3. Las acciones o targets en iptables

Cada cadena tiene que indicar la acción a realizar ocn el paquete en el caso de coincidencia:

  • ACCEPT acepta el paquete.

  • DROP rechaza el paquete sin ofrecer información.

  • REJECT rechaza el paquete ofreciendo información al remitente.

  • QUEUE envia el paquete al espacio de usuario donde puede ser tratado mediante scripts u otras herramientas no incluidas dentro del núcleo.

  • RETURN si es desde una cadena definida por el usuario, se continua el procesado del paquete desde la cadena que invocó a la cadena definida por el usuario. Si es desde una cadena predefinida se aplica la política por defecto de esa cadena al paquete.
