1. ¿Qué es PERL?


PERL (Practical Extraction and Report Languaje) es un lenguaje creado en 1987 por Larry Wall.

La versión 1.0 se lanzó el 18 de Octubre de ese mismo año en el grupo de usenet comp.sources, aunque hasta la versión 3.0 del 18 de Octubre de 1989 no se publica con licencia GNU.

El objetivo para el cuál se diseña Perl es el disponer de un lenguaje de propósito general que permita simplificar la mayoría de las tareas de administración de un sistema tipo UNIX.

Actualmente se encuentra por la versión 5.6 que es la que se puede encontrar en la mayoría de las distribuciones Linux actuales y se está trabajando en la 6.0. Perl es "multiplataforma", en el sentido de que se encuentra disponible para un amplio número de sistemas operativos como Linux, UNIX, MVS, VMS, MS/DOS, Macintosh, OS/2, Amiga
 
 

2. ¿Cuáles son las características de Perl?

Perl es interpretado. Al contrario que otros lenguajes como C/C++ o Pascal, los programas Perl son scripts de aspecto parecido a los scripts para la shell (aunque en la realidad el intérprete los compila antes de ejecutarlos)

Se tienen las estructuras de control de los lenguajes estructurados modernos como bloques condicionales, bloques iterativos, definición de rutinas, funciones y módulos, etc...

Perl nació en UNIX y por tanto pone a disposición del programador la mayoría de las llamadas que podemos encontrar en ese sistema. En otras plataformas alguna de las funcionalidades de esas llamadas puede no estar disponible. Entre las llamadas al sistema de que disponemos están las de manipulación de ficheros (open, read, write, ...), de creación de procesos (fork, exec, kill, ...) de comunicaciones (getnetbyaddr, socket, sokectpair, ...)

Tiene un mecanismo de  búsqueda de patrones y expresiones regulares extraordinariamente potente. Esto lo hace adecuado para tareas de administración de sistema, donde a veces es necesario tratar con grandes cantidades de información distribuida en múltiples ficheros (ficheros de log).

Hay mucha gente que programa en Perl, lo que nos permite encontrar en Internet una gran cantidad de código en forma de módulos y reutilizarlo para resolver un amplio abanico de problemas

Se puede aplicar a la tecnología CGI (Common Gateway Interface) y algunos programas lo están usando como lenguaje de scripts (perl-fu para Gimp).
 
 

3. ¿Cómo sé si dispongo de Perl en mi sistema?

Si la plataforma en la que se trabaja es UNIX-like, basta con teclear el comando en el prompt de la shell
 
$    which perl
y si se obtiene algo parecido a lo siguiente:
 
/usr/bin/perl
es que está instalado y accesible en el PATH

En otro tipo de plataformas basta con mirar en los directorios donde se suelen instalar los programas.
