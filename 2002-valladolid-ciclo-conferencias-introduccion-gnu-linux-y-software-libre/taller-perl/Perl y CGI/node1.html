1. ¿Qué es CGI?


CGI (Common Gateway Interface) es una tecnología usada en Internet para crear páginas web al instante basada en información recogida del usuario (generalmente via formularios).

Se basa en la capacidad de configurar los servidores web para que en vez de seguir la secuencia usual (un cliente web pide un fichero al servidor, y éste se lo manda usando el protocolo http), el fichero no se manda sino que se ejecuta en el propio servidor y la salida del proceso es la que se manda al cliente.

Naturalmente, han aparecido alternativas que en vez de procesar en el servidor (por la sobrecarga que puede producirse) se hace en el cliente (javascript, java - si bien este último ha introducido otra vez el antiguo esquema - sevlets).

Los scripts cgi se pueden hacer en muchos lenguajes, aunque el más utilizado es Perl por la capacidad que tiene para analizar transformar texto (además de que hay muchos módulos para conectar Perl con bases de datos).

El directorio dónde residen depende del servidor web utilizado. En el caso de apache, suelen estar en el directorio /var/www/cgi-bin o /usr/lib/cgi-bin aunque otros directorios pueden ser utilizados si se configura el apache para ello.
 
 
 

2. Secuencia de una sesión cgi en Perl

Cuando el programa cgi es un script de Perl, la secuencia se puede especificar un poco más:

1. El cliente realiza una petición estándar URL a un servidor http. No sólo se envía esto. También un encabezado con datos provistos por el cliente

2. Este encabezado se pasa al programa como variables de entorno $ENV{ }

3. Se invoca al intérprete de perl y se ejecuta el script

4. El script devuelve los datos de su procesamiento al servidor (generalmente en formato html)

5. El servidor se los envía al cliente
